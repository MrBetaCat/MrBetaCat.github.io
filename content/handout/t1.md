# T1 Handout — 最简 Agent 骨架 / Minimal Agent Skeleton
## Module 1: Minimal Skeleton

---

### Core Concept

An AI agent is not an LLM. It is an LLM plus a harness — a loop that perceives
inputs, reasons, selects actions, executes them, and feeds results back. The
minimal skeleton is the smallest harness that makes this loop runnable. Without
the loop, you have a model that responds once and stops. That is not an agent —
it is an expensive autocomplete. Every capability in the 30-topic curriculum is
an elaboration of one component in this loop.

---

### Car Metaphor

The agent loop is the chassis. The LLM is the engine.
The engine cannot move the car. The chassis cannot think. You need both —
but when your agent breaks, the problem is almost always in the chassis,
not the engine. Build the chassis explicitly. Never hide it inside a framework
until you can draw it from memory.

---

### Code Skeleton (the loop, annotated)

```python
messages = [{"role": "system", "content": SYSTEM}, {"role": "user", "content": question}]

for iteration in range(1, max_iterations + 1):     # ← brakes: chassis enforces stop
    response = ollama.chat(model=MODEL, messages=messages, tools=TOOLS)
    msg = response.message

    if not msg.tool_calls:                          # ← parse: no tool call = done
        return msg.content

    messages.append({"role": "assistant",           # ← observe step 1: log assistant turn
                     "content": msg.content or "",
                     "tool_calls": msg.tool_calls})

    for tool_call in msg.tool_calls:
        result = execute_tool(                      # ← execute: harness runs tool
            tool_call.function.name,
            tool_call.function.arguments)
        messages.append({"role": "tool",            # ← observe step 2: result enters context
                         "content": result,
                         "name": tool_call.function.name})

return "(loop guard fired)"                         # ← safety net: always reachable
```

Five components — four are harness code, one is model:
  perceive → build_context → [LLM] → parse_action → execute → observe

---

### Failure Modes

**Missing loop guard:** Removing `max_iterations` causes the model to call tools
indefinitely — no convergence, runaway token cost. Fix: `for iteration in
range(1, max_iterations + 1)` in harness code. Never in a prompt.

**Exception in execute_tool():** A raised exception kills the loop instantly with
no recovery. Fix: all tool functions return strings in every case — success or
failure. Errors are observations. Exceptions are reserved for infrastructure
failures only.

---

### Future AI Agent Design Decision #1

`execute_tool()` is a pure string-returning function with a top-level
`try/except`. `max_iterations=10` is a harness constant, not a prompt
instruction.
